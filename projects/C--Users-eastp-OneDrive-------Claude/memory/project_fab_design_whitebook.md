도요님이 업무 중 나오는 FAB 도메인 지식(벽체 종류, 기류, 방폭 등)을 별도로 정리해 "FAB 설계백서"로 쌓는 중. 업무목록(할 일 추적)과는 별개 — 순수 학습·레퍼런스 목적.

**저장 위치**: `Dongmin\40_Resources\FAB기술\` — 주제별로 파일을 나눠 저장(예: `CR_벽체_종류.md`).

**Why:** 2026-07-13 첫 항목(CR 벽체 3종: Bay Partition Wall / Contamination Wall / Low Partition Wall) 작성 후, 도요님이 "주제별로 나누고 나중에 MOC 만들자"고 방향 확정.

**How to apply:**
- 새 도메인 지식이 나올 때마다 주제 단위로 **별도 파일**을 만든다(한 파일에 다 몰아넣지 않음).
- MOC(지도 노트)는 지금 만들지 않는다 — 파일이 어느 정도 쌓인 뒤 나중에 일괄 생성. 매번 새 파일 만들 때 MOC를 갱신하려 하지 말 것.
- vault에 이미 `40_Resources\_MOC\` 폴더가 있으니, 나중에 MOC 만들 때 그 폴더 컨벤션을 따른다.

**2026-07-14 업데이트:** 전용 스킬 `/설계백서` 생성(`C:\Users\eastp\.claude\skills\설계백서\SKILL.md`). 내용 입력 → 관련 법규·기준 검토 → 보완할 부분 사용자에게 먼저 질문 → md 저장 순서로 동작. 사용자 확인 없이 임의로 완성해서 저장하지 않는 게 핵심 규칙.
