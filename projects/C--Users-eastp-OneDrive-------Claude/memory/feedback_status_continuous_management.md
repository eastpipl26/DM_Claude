업무목록.md 상태 컬럼은 기존 완료/진행/지연/보류/대기 외에 **🔵지속관리**를 추가로 쓴다.

**Why:** ID5(Y1 마감/미마감 자료 공유)처럼 공사 진행에 따라 계속 버전이 갱신돼야 하는 항목은 "진행"이라 하기도, "완료"라 하기도 애매함. 도요님이 2026-07-13에 "이건 원래 지속적으로 버전관리를 해야하는거라서 별도 스테이터스가 있어도 좋을듯"이라고 제안.

**How to apply:** 한 번의 완료 시점이 없고 지속적으로 갱신·재접수·재배포되는 자료(마감/미마감 현황, 버전관리 도면 등)는 상태를 🔵지속관리로 표기한다. 완료 여부를 억지로 판단하지 않는다.

**상태 전환**: 🔵지속관리 항목에 도요님이 직접 대응해야 할 이슈(추가 의견 검토 등)가 생기면 🟡진행으로 전환하고, 대응이 끝나면 다시 🔵지속관리로 돌아간다. 완료(✅)는 이 항목 자체가 더 이상 버전관리가 필요 없어지는 시점(예: 최종 확정)에만 부여한다.
