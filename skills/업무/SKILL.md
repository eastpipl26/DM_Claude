# 업무 스킬

오늘/이번주까지 처리해야 하는 SK하이닉스 본업 업무를 마감일 기준으로 조회한다. 등록·수정은 `업무-관리` 스킬 담당, 이 스킬은 조회 전용.

## 사용법
```
/업무 오늘
/업무 이번주
```
스킬 호출 없이 "오늘까지 해야하는 일 뭐야", "이번주까지 할 일 알려줘" 같은 자연어로 물어도 동일하게 처리한다.

## 데이터 위치 (config.md 참조)

`sk_work` = `{sk_hynix}\업무` 아래, 항목별 개별 md 파일 방식(2026-07-14부터):
- `items\*.md` — 업무 항목 1건당 1파일. frontmatter: id·구분·제목·대응처·담당자·마감일·완료일·카테고리·상태·등록일. 카테고리는 다중 태그 리스트(2026-07-15 추가, 필터용) — 법규/간섭/설계변경/시공성/구조/비용/BIM관리/인허가/LL/협의/일정 등, 항목에 여러 개 붙을 수 있고 필요시 새 카테고리 추가 가능. 본문은 `## 메모 로그` 아래 시간순 불릿.
- `인허가_items\*.md` — 인허가 건 1건당 1파일. frontmatter: id·건명·기관·신청일·처리기한·상태·다음액션. 본문은 메모 로그.
- `업무현황.base` / `인허가현황.base` — Obsidian Base 뷰(표시·필터용, 조회는 이 스킬이 md를 직접 읽어 처리).
- (구버전 `업무목록.md`/`인허가.md`는 `_구표형식_백업.md`로 보관, 참조하지 않음)

## 동작

1. **"오늘"**: `items\`·`인허가_items\` 각 md의 frontmatter에서 마감일/처리기한 == 오늘 날짜인 항목을 찾는다. 이미 지난 날짜인데 상태가 완료가 아닌 항목(연체)도 함께 표시한다.
2. **"이번주"**: 이번주 월요일~일요일 기준. 오늘부터 이번주 일요일까지 마감일이 있는 항목을 찾는다(주는 월요일 시작, 일요일 종료).
3. **제외 대상**: 상태가 ✅완료인 항목은 제외. 🔵지속관리 항목은 마감일이 없는 경우가 많아 보통 제외되지만, 마감일이 박혀 있으면 포함.
4. **정렬**: 마감일 임박순(연체 → 오늘 → 이후).
5. **출력**: 항상 표 형식으로 응답한다 — ID·제목·대응처·담당자·마감일·상태·비고(메모 마지막 로그 한 줄 요약). 채팅 텍스트로 줄글 나열하지 않는다.
6. 파일 수정 없음 — 순수 조회.

## 하지 말 것

- 마감일 없는 항목을 임의로 "오늘 할 일"에 넣지 않는다.
- 완료 항목을 목록에 포함하지 않는다.
