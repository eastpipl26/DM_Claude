# 블로그-작성 스킬

## 사용법
```
/블로그-작성 <폴더경로 또는 파일경로들> [주제]
```
예시:
```
/블로그-작성 C:\Users\eastp\Pictures\더현대\
/블로그-작성 C:\Users\eastp\Pictures\IMG_001.jpg C:\Users\eastp\Pictures\IMG_002.jpg 주말 나들이
```

---

## 동작 순서

### 1단계: 입력 파싱
`$ARGUMENTS`에서 경로와 주제를 파싱한다.
- 폴더 경로면 → 해당 폴더의 이미지 파일 전체 목록 수집 (jpg, jpeg, png, webp 대상)
- 파일 경로 목록이면 → 그 파일들만 처리
- 마지막 인자가 경로 형식이 아니면 → 주제로 간주
- 주제가 없으면 사진 분석 후 자동 추정

HEIC 파일은 읽을 수 없으니, 발견되면:
"[파일명].HEIC 파일은 읽을 수 없습니다. 아이폰 설정 > 카메라 > 포맷 > 호환성 우선으로 바꾸거나 JPG로 변환 후 다시 시도해주세요." 라고 안내하고 해당 파일은 건너뛴다.

### 2단계: 이미지를 Obsidian vault로 복사
사진 분석 전, 이미지 파일을 vault attachment 폴더로 복사한다.

- 복사 대상 폴더: `C:\Users\eastp\iCloudDrive\Dongmin\90_System\Attachments\블로그\<YYMMDD_키워드>\`
  - 키워드는 원본 폴더명 또는 주제에서 간단히 추출
- PowerShell 명령으로 실행:
  ```
  New-Item -ItemType Directory -Force "C:\Users\eastp\iCloudDrive\Dongmin\90_System\Attachments\블로그\<폴더명>"
  Copy-Item "<원본 이미지 경로들>" "위 폴더"
  ```
- 복사 완료 후 파일명 목록 기억 (글 작성 시 내부링크에 사용)

### 3단계: 사진 분석
각 이미지를 Read 툴로 읽고, 사진마다 아래를 메모한다:
- 장소 / 활동 / 주요 피사체
- 분위기 (밝은, 아늑한, 활기찬 등)
- 글에서 어울릴 위치 (도입, 중반, 마무리)
- 한 줄 설명 (네이버 블로그 캡션용)

### 4단계: 글 작성
**작성자 관점 — 도요(아빠) 기준:**
- 글쓴이는 항상 **아빠(도요)** 시점. "저는", "제가", "아이" 등 아빠 1인칭.
- "남편이" 표현 절대 사용 금지 — 도요 본인이 쓰는 글이므로.
- 아이와 함께한 경험, 아빠 육아 시점의 솔직한 감정을 담는다.
- 예: "제가 직접 분유를 먹여봤는데…", "아이를 재우다가 발견한 건데요…"

**네이버 블로그 스타일 기준:**
- 친근한 구어체 (저는, 제가, ~했어요, ~네요, ~거든요)
- 짧은 문단 — 3~5줄마다 줄바꿈
- 도입부: 날씨, 기분, 계기로 자연스럽게 시작
- 사진 전후: 해당 사진과 연결되는 문장 배치
- 마무리: 소감 or 다음 기약으로 마무리
- 이모지: 자연스럽게, 과하지 않게 (문단당 1개 정도)
- 제목: 장소/활동 키워드 포함, 짧고 명확하게 (예: "더현대 서울, 주말 나들이 후기 ☀️")

**글 구조 — 소제목으로 위계 만들기:**
- 글 전체를 2~4개 섹션으로 나눈다. 섹션마다 소제목을 붙인다.
- 소제목 형식:
  - Obsidian MD: `## 소제목` (H2)
  - 네이버 복붙 포맷: `▶ 소제목` (굵게 처리 안내 포함)
- 소제목 예시: "📦 제품 첫인상", "👨 아빠도 써봤어요", "🛒 같이 구매한 것들", "✨ 총평"
- 섹션 흐름: 도입 → 제품 소개 → 사용 경험 → 마무리 순이 기본. 내용에 따라 조정.

**글 길이**: 사진 수 × 150~200자 정도 (사진 3장이면 450~600자)

### 5단계: 두 가지 포맷 생성

**포맷 A — Obsidian MD** (저장용):
```markdown
---
title: <제목>
date: <오늘 날짜 YYYY-MM-DD>
type: 블로그
tags: [블로그, <주제태그>]
status: draft
---

## <소제목1>

<글 내용>

![[<파일명.jpg>]]
*<사진 캡션>*

## <소제목2>

<이어지는 글>
```
→ `file://` 절대경로 대신 Obsidian 내부링크 `![[파일명]]` 사용
→ 파일명은 2단계에서 vault로 복사한 파일명 그대로
→ 소제목은 `##` H2 레벨 사용

**포맷 B — 네이버 복붙 포맷** (붙여넣기용):
```
━━━━━━━━ 네이버 블로그 복붙 포맷 ━━━━━━━━

제목: <제목>

▶ <소제목1>
(네이버 에디터에서 이 줄을 굵게/크게 처리해주세요)

<글 내용>

📷 [사진1 삽입]
  파일: <파일명>
  내용: <사진 설명>

▶ <소제목2>
(네이버 에디터에서 이 줄을 굵게/크게 처리해주세요)

<이어지는 글>

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
※ 사진은 위 📷 표시 위치에 직접 업로드해주세요.
※ ▶ 소제목 줄은 네이버 에디터에서 굵게 또는 글씨 크기 키워서 강조해주세요.
```

### 6단계: 옥순이에게 저장 위임
아래 내용으로 옥순이(obsidian-manager 에이전트)에게 위임한다:
- 저장 경로: `C:\Users\eastp\iCloudDrive\Dongmin\30_Areas\블로그\<파일명>.md`
- 파일명: `YYMMDD_<주제키워드>.md` (공백 없이, 한글 가능)
- 폴더 없으면 생성 후 저장
- 저장 완료 후 경로 보고

### 7단계: 사진 폴더 자동 열기
아래 PowerShell 명령을 실행해 vault 내 사진 폴더를 탐색기에서 자동으로 연다:
```
explorer.exe "C:\Users\eastp\iCloudDrive\Dongmin\90_System\Attachments\블로그\<YYMMDD_키워드>"
```

### 8단계: 완료 보고
말숙이가 사용자에게 보고한다:
- 이미지 복사 경로 (90_System/Attachments/블로그/...)
- 저장된 블로그 파일 경로
- 네이버 복붙 포맷을 터미널에 출력
- 안내 메시지: "사진 폴더가 탐색기에서 열렸습니다. 네이버 에디터에 글 붙여넣기 후, 📷 번호 순서대로 폴더에서 사진을 드래그앤드롭해주세요."
