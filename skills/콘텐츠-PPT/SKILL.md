# 콘텐츠-PPT 스킬

## 사용법

```
/콘텐츠-PPT <주제 텍스트>
/콘텐츠-PPT <기존 파일 경로>

예시:
/콘텐츠-PPT 토스트샌드 여름 신제품 소개
/콘텐츠-PPT C:\Users\eastp\iCloudDrive\Dongmin\30_Areas\블로그\260628_토스트샌드.md
```

---

## 말숙이 동작: 파싱 후 즉시 위임

말숙이는 아래 3단계만 수행한다. 실제 작업은 줄이 → 채린이 → 판이가 전담한다.

### 1단계: 입력 파싱

입력이 파일 경로(.md)인지 주제 텍스트인지 판별한다.
- `.md` 확장자 또는 실제 경로 형태 → 파일 경로
- 그 외 → 주제 텍스트

### 2단계: 줄이 → (채린이) → 판이 순서로 위임

**줄이에게 먼저 위임 (`run_in_background: true`):**
- 주제 텍스트 또는 파일 경로 전달
- 슬라이드 단위 대본 작성 + vault 저장
- **줄이는 작업 완료 후 판이를 직접 호출한다** (말숙이 재개입 없음)
  - 줄이 → 판이 체인은 줄이 내부에서 처리
  - 채린이 테마도 줄이가 판이에게 전달할 때 함께 넘김

**채린이에게 병렬 위임 (`run_in_background: true`):**
- "유튜브/SNS 콘텐츠 PPT용 Marp CSS 테마 준비" 요청
- 기본 테마: `dark-card-theme.css` (다크 배경·카드형 비교 레이아웃·페이드인 애니메이션).
  주제 톤이 안 맞으면(예: 밝은 톤 요청) 채린이가 대체 테마를 새로 만든다.
- 완료 시 테마 경로를 bus 또는 직접 판이에게 전달

**판이는 줄이가 직접 호출:**
- 말숙이가 아닌 줄이가 판이를 spawn
- 이렇게 해야 말숙이가 체인 중간에도 자유롭게 명령 수신 가능

### 3단계: 완료 보고

판이 완료 보고 수신 후 도요님께 결과 전달:
- 대본 파일 경로
- HTML 프레젠테이션 경로 (브라우저로 열면 방향키·스페이스로 장표 전환)
- 슬라이드 수
- 적용 테마명

말숙이는 대본 작성, PPT 변환, 파일 저장을 직접 수행하지 않는다.
