# 업무-관리 스킬

SK하이닉스 본업 업무 대시보드(일일·긴급·추적·마감·인허가) 관리.

## 사용법
```
/업무-관리 <자연어 지시>
```
예시:
```
/업무-관리 긴급 추가: F02 CR층 도면 회신, 발주처, 목요일까지
/업무-관리 3번 완료
/업무-관리 인허가 추가: Y1P4 허가변경, 용인시, 기한 7/21
/업무-관리 현황
```
스킬 호출 없이 "긴급 하나 추가해줘", "대시보드 갱신해줘" 같은 자연어로 말해도 동일하게 처리한다.

---

## 데이터 위치 (config.md 참조, 2026-07-14부터 개별 md 방식)

`sk_work` = `{sk_hynix}\업무` 아래:
- `items\{ID}_{제목슬러그}.md` — 업무 항목 1건당 1파일. frontmatter: id·구분(일일/긴급/추적/일반)·제목·대응처(발주처/CM/공사팀/공무팀/인허가/내부/개인)·담당자·마감일·완료일·카테고리·상태(⏳대기/🟡진행/✅완료/⏸보류/🔴지연/🔵지속관리)·등록일. 본문 `## 메모 로그`에 시간순 불릿(`[MM-DD H시] 내용`) 누적.
  - **카테고리** (2026-07-15 추가): 리스트형 다중 태그, Base 필터용. 기본 태그: 법규/간섭/설계변경/시공성/구조/비용/BIM관리/인허가/LL/협의/일정. 항목 성격에 맞게 여러 개 부여 가능하고, 기존 태그로 안 맞으면 새 카테고리를 만들어도 된다. 신규 항목 등록·기존 항목 수정 시 항상 이 필드를 채운다(비워두지 않음).
- `공사일정.md` — 항목·구분(설계/공사/인허가/개인)·시작일·마감일·상태 (당분간 표 형식 유지)
- `인허가_items\{ID}_{건명슬러그}.md` — 인허가 건 1건당 1파일. frontmatter: id·건명·기관·신청일·처리기한·상태(준비/신청/보완/회신대기/승인/반려)·다음액션. 본문 메모 로그.
- `업무현황.base` / `인허가현황.base` — Obsidian Base 뷰. 새 항목 추가 시 파일만 생성하면 Base에 자동 반영됨(별도 등록 불필요).

## 동작

1. **파싱**: 지시에서 동작(추가/완료/변경/삭제/현황/갱신)과 대상을 판별한다.
   - "긴급/일일/추적/업무" → `items\` · "일정/마감/마일스톤" → 공사일정.md · "인허가/허가/신고" → `인허가_items\`
   - 상대 날짜("목요일까지", "다음주 월요일")는 오늘 기준 YYYY-MM-DD로 변환한다.
2. **추가**: `items\` 또는 `인허가_items\` 폴더에 새 md 파일 생성. ID는 기존 파일명 중 최대값+1. 파일명 `{ID:03d}_{제목 슬러그}.md`. frontmatter 채우고 본문에 `## 메모 로그` + 첫 로그 항목(`[MM-DD H시] 등록. ...`) 작성.
3. **완료/변경**: 대상 파일을 찾아(제목·ID로 매칭, 모호하면 확인 후 진행) frontmatter 상태 필드 수정 + 본문에 로그 한 줄 추가. 기존 로그는 지우지 않는다.
4. **대시보드 재생성**: 필요시 `python {work_scripts}\build_work_dashboard.py` 실행 (config.md의 `work_scripts` 참조). Base 뷰는 파일 저장만으로 자동 갱신되므로 필수는 아님.
5. **보고**: 반영 파일 경로 + 현재 긴급/마감임박 건수 요약.
6. **현황 요청 시**: `items\`·`인허가_items\`의 md를 읽고 긴급 → 마감임박(D-7) → 오늘 할 일 순으로 채팅에 요약한다. 파일 수정 없음.

## 운영 규칙 (2026-07-16 도입, RFI·GTD 관행 기반)

1. **⏳대기 = 회신기한 필수**: 상태를 ⏳대기로 바꿀 때 마감일에 회신기한 기입(미지정 시 +7일). 기한 도과 시 F/U 로그 남기고 기한 재설정 — F/U 이력은 매번 기록(지연 책임 방어용).
2. **[결정] 태그**: 의사결정 사항은 `- [MM-DD] [결정] 내용 — 사유: ...` 형식으로 기록. 경과 로그와 구분.
3. **월요일 주간 리뷰**: 매주 월요일 미결 항목 전수 스캔 → 기한도과 대기 F/U 목록·마감지난 진행 재분류·날짜없는 "확인 필요" 배치 → 당일 데일리노트 체크리스트 반영. (정기 태스크 weekly-work-review로 자동화)
4. **신규 등록 전 중복 확인**: 관련 기존 항목 후보를 검색해 2개 이상 유사하면 "신규 vs 기존 추가"를 도요님께 확인.

## 회의록·이슈 연계 (자동)

회의록(meeting-notes)이나 이슈(issue-register) 정리가 끝나면, 액션아이템 중 **도요가 직접 해야 하거나 기한이 있는 것**을 업무목록.md에 자동 등록하고 대시보드를 재생성한다. 등록 시 메모에 출처 회의록 파일명을 남긴다.

## 하지 말 것

- 완료 처리는 행 삭제가 아니라 상태=완료로 변경 (기록 보존).
- 확실하지 않은 마감일을 추측으로 넣지 않는다 — 기한 불명이면 마감일 공란.
