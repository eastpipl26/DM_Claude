# 업무-관리 스킬

SK하이닉스 본업 업무 대시보드(일일·긴급·추적·마감·인허가) 관리.

## 사용법
```
/업무-관리 <자연어 지시>
```
예시:
```
/업무-관리 긴급 추가: F02 CR층 도면 회신, 발주처, 목요일까지
/업무-관리 3번 완료
/업무-관리 인허가 추가: Y1P4 허가변경, 용인시, 기한 7/21
/업무-관리 현황
```
스킬 호출 없이 "긴급 하나 추가해줘", "대시보드 갱신해줘" 같은 자연어로 말해도 동일하게 처리한다.

---

## 데이터 위치 (config.md 참조)

`sk_work` = `{sk_hynix}\업무` 아래 3개 원본 md:
- `업무목록.md` — ID·구분(일일/긴급/추적/일반)·제목·대응처(발주처/CM/공사팀/공무팀/인허가/내부/개인)·마감일·상태(대기/진행/완료/보류)·등록일·메모
- `공사일정.md` — 항목·구분(설계/공사/인허가/개인)·시작일·마감일·상태
- `인허가.md` — 건명·기관·신청일·처리기한·상태(준비/신청/보완/회신대기/승인/반려)·다음 액션

## 동작

1. **파싱**: 지시에서 동작(추가/완료/변경/삭제/현황/갱신)과 대상 표를 판별한다.
   - "긴급/일일/추적/업무" → 업무목록.md · "일정/마감/마일스톤" → 공사일정.md · "인허가/허가/신고" → 인허가.md
   - 상대 날짜("목요일까지", "다음주 월요일")는 오늘 기준 YYYY-MM-DD로 변환한다.
2. **md 갱신**: 해당 표에 행 추가/수정. ID는 기존 최대값+1. 등록일은 오늘. 예시 행(`예시 행` 메모)이 남아 있으면 첫 실제 등록 때 삭제한다.
3. **대시보드 재생성**: `python {work_scripts}\build_work_dashboard.py` 실행 (config.md의 `work_scripts` 참조).
4. **보고**: 반영 내용 + 대시보드 경로(`{sk_work}\업무대시보드.html`) + 현재 긴급/마감임박 건수 요약.
5. **현황 요청 시**: md를 읽고 긴급 → 마감임박(D-7) → 오늘 할 일 순으로 채팅에 요약한다. 파일 수정 없음.

## 회의록·이슈 연계 (자동)

회의록(meeting-notes)이나 이슈(issue-register) 정리가 끝나면, 액션아이템 중 **도요가 직접 해야 하거나 기한이 있는 것**을 업무목록.md에 자동 등록하고 대시보드를 재생성한다. 등록 시 메모에 출처 회의록 파일명을 남긴다.

## 하지 말 것

- 완료 처리는 행 삭제가 아니라 상태=완료로 변경 (기록 보존).
- 확실하지 않은 마감일을 추측으로 넣지 않는다 — 기한 불명이면 마감일 공란.
