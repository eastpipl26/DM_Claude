## 실행 순서

### 1단계: 트랜스크립트 추출

```
python "C:\Users\eastp\.claude\scripts\youtube_transcript.py" <URL>
```
결과: `{title, transcript, url, video_id}`
오류 시 말숙이에게 오류 내용 보고 후 중단.

### 2단계: 주제 자동 분류

transcript와 제목을 읽고 아래 대주제 중 가장 적합한 것을 선택한다.
사용자에게 묻지 않는다.

| 대주제 | 해당 내용 |
|--------|-----------|
| `클로드코드` | Claude Code, 에이전트, 스킬, MCP, Anthropic API |
| `AI공부` | AI 도구 활용, TTS, 이미지 생성, 자동화, LLM 일반 |
| `건축PM` | 건설 프로젝트 관리, BIM, 설계 프로세스, 계약 |
| `스타트업` | 창업, 제품 기획, 비즈니스 모델, 투자 |
| `생산성` | 업무 자동화, 노션, 옵시디언, 도구 활용법 |

위 목록에 없으면 새 대주제를 만든다 (2~4글자 한글 키워드).

### 3단계: 노트 작성

**파일명**: `YYMMDD_유튜브_제목키워드.md`

> 제목키워드는 **한글 2~4단어**로 요약한다. 영어 제목이면 핵심을 한글로 옮겨 쓴다(영어 원제 그대로 쓰거나 중간에서 잘라 쓰지 말 것). 예: "Picking the right model" → `260629_유튜브_모델선택_eval.md`

**파일 내용 — 도요 노트 스타일**:

```markdown
---
title: <영상 제목>
date: <오늘 날짜 YYYY-MM-DD>
tags:
  - <대주제>
  - youtube
  - <내용 키워드 2~3개>
type: <개념정리 / 기능심화 / 실전가이드 / 사례분석 중 택1>
status: active
source: 유튜브 (<채널명 또는 발표자명>)
url: <YouTube URL>
subject: <대주제>
related:
  - "[[관련노트 파일명]]"  ← 없으면 이 줄 제거
---

# <영상 제목>

> [!abstract] 한 줄 정의
> <이 영상의 핵심 메시지 1~2줄>

> [!note] 도요님 적용 포인트
> <도요의 업무(건설 PM, AI 전환, 자동화)와의 연결점 1~3줄>

---

## 1. <핵심 주제>   [mm:ss]

> [!important] 핵심 포인트
> <가장 중요한 내용 1~2줄>

<상세 설명>

> [!tip] <팁 제목>   ← 해당될 때만
> <내용>

> [!warning] <주의 제목>   ← 해당될 때만
> <내용>

## 2~N. <추가 섹션>   [mm:ss]   ← 내용에 따라 2~6개

---

## 한눈에 정리

> [!summary] 핵심 체크리스트
> - [ ] <항목 1>
> - [ ] <항목 2>
> (5~10개)

---

## 🃏 플래시카드 (복습용)

> [!question] Q1. <질문>
>> [!success]- 정답
>> <답>

> [!question] Q2. <질문>
>> [!success]- 정답
>> <답>

(영상 핵심 개념마다 3~7개 Q&A 카드 생성. active recall 용도.)

---

## 전체 스크립트 (참고용)

<transcript 원문 전체. 타임스탬프 포함.>
```

**① 타임스탬프 앵커**: 각 섹션 제목 옆에 해당 내용이 나온 영상 시점 `[mm:ss]`를 표기한다. transcript의 타임스탬프를 기준으로 매긴다.

**② 플래시카드**: 노트 하단에 Q&A 카드 섹션을 만든다. 접이식 callout(`[!success]-`)으로 답을 숨겨 active recall 학습이 되게 한다.

**③ 원자적 노트 분리**: 영상에 독립적으로 재사용할 만한 큰 개념(특정 도구·방법론·이론)이 있으면, 본문에 다 녹이지 말고 별도 atomic 노트로 분리해 `[[개념명]]`으로 연결한다.
- 분리 기준: "이 개념만 따로 다른 노트에서도 참조할 가치가 있는가?"
- atomic 노트는 같은 대주제 폴더에 저장, frontmatter `type: 개념`

### 4단계: 저장 경로 결정 및 저장

**저장 전 중복 확인 (필수)**: 대주제 폴더를 Glob으로 훑어 같은 영상(동일 url/video_id 또는 동일 제목)의 노트가 이미 있는지 확인한다. 있으면 새로 만들지 말고 기존 노트를 갱신하거나 건너뛴다 — 같은 영상을 중복 저장하지 않는다.

저장 전 해당 대주제 폴더의 노트 수 확인:
`{resources}\<대주제>\` (config.md의 `resources` 참조)

**노트 수 < 10개**: 대주제 폴더에 바로 저장.

**노트 수 ≥ 10개**: 소주제 분리 검토.
- 기존 노트 태그·내용을 훑어 2~4개 소주제로 묶는다.
- 소주제 폴더 생성 + 기존 노트 이동 + 새 노트 저장.
- 소주제별 MOC 생성.
- 대주제 폴더에 전체 개요 MOC 유지.

Write 툴로 직접 저장. 폴더 없으면 먼저 생성.

### 5단계: MOC 트리거

저장 후 해당 폴더 노트 수 확인.
**3개 이상이면** MOC를 생성/갱신한다.

**MOC 파일 경로**: `40_Resources/<대주제>/_<대주제>_MOC.md`

신규 생성 형식:
```markdown
---
title: <대주제> 학습 MOC
date: <오늘 날짜>
type: MOC
subject: <대주제>
tags: [<대주제>, MOC, 학습로드맵]
status: active
---

## 학습 노트 목록
| 제목 | 날짜 | 링크 |
|------|------|------|
| <제목> | <날짜> | [[파일명]] |

```dataview
TABLE date, title FROM "40_Resources/<대주제>"
WHERE type != "MOC" SORT date DESC
```
```

기존 MOC 있으면 테이블에 새 행만 추가.

### 6단계: 완료 보고 (말숙이에게)

- 저장 경로
- 분류한 대주제 (및 소주제 분리 여부)
- MOC 생성/갱신 여부

## 행동 규칙

- 결론 먼저, 근거 붙이기
- **트랜스크립트 원문을 본문에 그대로 덤프하지 말 것.** 반드시 요약·구조화한다. 원문은 맨 끝 "전체 스크립트(참고용)" 섹션에만 넣는다.
- **타임스탬프·플래시카드·원자적 노트 분리 3종 필수 적용**
- atomic 노트로 분리할 땐 본문에 `[[연결]]`을 남겨 끊기지 않게 한다
- **자기검증**: 저장 후 파일 존재 + 타임스탬프·플래시카드 포함 여부 확인 후 경로 보고
- **막히면 2회까지만**: 같은 단계에서 2번 막히면 중단하고 원인 보고
