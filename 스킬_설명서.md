# 스킬(Skill) 설명서

> 한 줄 요약: 스킬은 **자주 쓰는 업무를 `/명령어` 하나로 실행하는 자동화 레시피**다.

---

## 스킬이 뭔가

Claude에게 "회의록 정리해줘"라고 매번 설명하는 대신, `/회의록-정리` 하나만 치면 Claude가 정해진 순서대로 알아서 처리하는 것. 반복 업무를 한 번 정의해두고 계속 재사용하는 구조다.

엑셀 매크로를 한 번 만들어두고 버튼 하나로 실행하는 것과 같은 개념이다.

---

## 스킬의 종류

### 1. 기본 스킬 (Anthropic 제공)

설치 없이 처음부터 있는 스킬들이다.

| 스킬 | 하는 일 |
|------|---------|
| `docx` | Word 문서 생성·편집 |
| `pdf` | PDF 처리 (생성, 추출, 병합) |
| `pptx` | 파워포인트 생성·편집 |
| `xlsx` | 엑셀 생성·편집 |
| `learn` | 개념 설명 (교육 모드) |
| `schedule` | 반복 자동화 예약 |

### 2. 플러그인 스킬 (설치 후 사용)

플러그인을 설치하면 추가되는 스킬들이다. 도요님은 현재 **Operations**와 **Product Management** 플러그인이 설치돼 있다.

#### Operations 플러그인 스킬
| 스킬 | 하는 일 | 언제 쓰나 |
|------|---------|----------|
| `operations:process-doc` | 프로세스 문서화 (플로우차트, RACI, SOP) | 업무 절차를 문서로 남길 때 |
| `operations:change-request` | 변경관리 요청서 작성 | 설계 변경·공정 변경 요청할 때 |
| `operations:status-report` | 상태 보고서 생성 | 주간보고·현안 보고서 만들 때 |
| `operations:risk-assessment` | 리스크 평가·완화 방안 | 리스크 목록 정리할 때 |
| `operations:capacity-plan` | 인력·자원 용량 계획 | 인원 배분, 과부하 점검할 때 |
| `operations:runbook` | 반복 작업 절차서 작성 | 정기 점검, 인계 문서 만들 때 |
| `operations:vendor-review` | 업체 평가·검토 | 협력사 선정·계약 갱신할 때 |

#### Product Management 플러그인 스킬
| 스킬 | 하는 일 | 언제 쓰나 |
|------|---------|----------|
| `product-management:stakeholder-update` | 이해관계자 보고 문서 | 발주처·임원 보고 만들 때 |
| `product-management:sprint-planning` | 단기 계획 수립 | 주간·월간 업무 계획 잡을 때 |
| `product-management:write-spec` | 스펙 문서 작성 | 설계 요건 정의서 만들 때 |
| `product-management:roadmap-update` | 일정·우선순위 재조정 | 공정 일정 바뀔 때 |
| `product-management:risk-assessment` | 리스크 식별·대응 | 프로젝트 리스크 관리할 때 |

### 3. 직접 만드는 스킬

`skill-creator` 스킬을 쓰면 나만의 스킬을 만들 수 있다. 예: `/회의록-정리`, `/이슈-등록` 같은 도요님 전용 스킬.

---

## 어떻게 쓰나

### 방법 1: 슬래시 명령어로 직접 실행
채팅창에 `/` 를 치면 사용 가능한 스킬 목록이 뜬다. 원하는 것을 클릭하거나 이름을 직접 입력한다.

```
/operations:status-report
→ Claude가 보고서 작성에 필요한 정보를 물어보고 자동으로 만들어준다.
```

### 방법 2: 자연어로 말하기
스킬 이름을 몰라도 된다. 그냥 하고 싶은 일을 말하면 Claude가 맞는 스킬을 알아서 연결한다.

```
"이번 주 설계 검토 회의 결과로 주간보고 만들어줘"
→ operations:status-report 스킬이 자동 실행된다.
```

---

## 스킬 vs 그냥 Claude에게 말하기

| | 스킬 사용 | 그냥 요청 |
|--|-----------|-----------|
| 결과 일관성 | 매번 같은 형식·순서 | 그때그때 다름 |
| 설명 필요 여부 | 처음에 한 번만 | 매번 설명 필요 |
| 적합한 경우 | 반복 업무 (보고서, 회의록) | 일회성 작업 |

## 앞으로 추가할 스킬 (도요님 전용)

CLAUDE.md에 명시된 업무 기반으로 만들 예정인 스킬들:

- `/회의록-정리` — 회의 내용 → 정리된 회의록
- `/회의록-요약` — 긴 회의록 → 핵심 요약
- `/이슈-등록` — 현안 입력 → 이슈 문서 생성
- `/이슈-현황` — 현재 이슈 목록 정리
