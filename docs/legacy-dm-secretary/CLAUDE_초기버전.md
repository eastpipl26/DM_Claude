## 행동 규칙 (실제 마찰에서 추가)

> 이 섹션은 처음부터 다 채우지 않는다. 작업하다가 클로드가 반복적으로 하는
> 거슬리는 행동을 발견할 때마다 한 줄씩 추가한다. 그래야 진짜 규칙이 된다.

- **첫 패스에서 끝까지 본다.** 분석·검토를 부탁하면 관련 파일을 전부 읽는다.
  피상적으로 훑고 답하지 않는다. 범위가 불확실하면 얕은 결과를 내놓지 말고 먼저 물어본다.
- **막히면 루프 돌지 말 것.** 같은 단계에서 2번 막히면 다시 시도하지 말고,
  무엇이 막혔는지 설명한다. 탐색·계획만 길게 하지 말고 구체적 결과물을 먼저 낸다.
- **끝나면 확인한다.** 요청한 작업을 마치면 묻지 않아도 알려준다:
  바꾼 파일 목록, (해당되면) 저장·동기화 여부.

## 명령어 / 스킬 네이밍 컨벤션

> slash 명령어나 스킬이 많아지면 접두사로 분류한다. 안 그러면 뒤죽박죽된다.

| 접두사       | 용도                          | 예시                          |
| ------------ | ----------------------------- | ----------------------------- |
| `회의록-*`   | 회의록 정리·요약              | `회의록-정리`, `회의록-요약`  |
| `이슈-*`     | 이슈 등록·추적                | `이슈-등록`, `이슈-현황`      |
| (접두사 없음)| 범용 작업                     | `저장`, `검색`                |

새 명령어를 만들 땐 맞는 접두사를 고른다. 기존 분류로 안 되는 경우에만 새 접두사를 만든다.
