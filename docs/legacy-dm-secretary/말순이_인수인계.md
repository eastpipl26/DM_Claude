## 반복하지 말아야 할 실수

### 파일·스크립트
- **스킬 name 필드**: 반드시 영문 kebab-case (`meeting-notes` ✓, `회의록-정리` ✗). 설치 시 YAML 검증에서 실패함.
- **파일 복사 시 인코딩**: 한글 파일을 bash cp로 복사하면 UnicodeDecodeError. Python으로 읽고 쓸 때 `errors='ignore'` 옵션 필수.
- **NTFS 파일 삭제**: sandbox(Linux)에서 `rm`으로 Windows 마운트 파일 삭제 불가. 사용자에게 직접 삭제 요청.
- **git 명령어**: sandbox에서 `git` 실행 시 `fatal: unknown error occurred while reading the configuration files`. git은 반드시 사용자 PowerShell에서 실행.
- **git-push.bat**: `cd /d 절대경로`는 PowerShell에서 파싱 오류. `pushd "%~dp0"` 사용.
- **git commit -m에 %date% %time%**: 한국어 날짜 형식이 포함되면 파싱 오류. 단순 문자열 사용.

### Chrome 자동화 (Google Calendar)
- **navigate 후 즉시 JS 실행 금지**: 페이지 로드 전이라 실패. 반드시 `find()`로 로드 확인 후 실행.
- **가져오기/내보내기 URL 직접 이동 불가**: 설정 페이지로 이동 후 `[role="treeitem"]`의 '가져오기' 항목 클릭.
- **캘린더 드롭다운**: `<select>` 아님. `[aria-haspopup="listbox"]` 클릭 후 `[role="option"]` 선택.
- **React 입력 필드**: `form_input` 도구 동작 안 함. `document.execCommand('insertText', false, '값')` 사용.
- **캘린더 신규 생성**: 자동화 시 자주 실패. 사용자가 직접 만드는 게 빠름.

### 스케줄 태스크
- **승인 창**: 태스크 생성 시 사용자 승인 팝업이 뜸. 이전 세션에서 inbox-watcher는 거절됨. 다시 생성 시 미리 설명하고 진행.
- **네트워크**: sandbox에서 외부 API(Telegram 등) 호출 불가 (403). 태스크는 Windows 환경에서 실행되므로 실제론 동작함.
