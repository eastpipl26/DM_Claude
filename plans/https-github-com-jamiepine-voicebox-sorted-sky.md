# 플랜: 개인비서 에이전트팀 고도화 + 모바일/작업환경 + AI 수익화 로드맵

## Context

도요(서동민)의 요청은 4가지다: ① 개인비서 에이전트팀을 꾸리는 데 필요한 "설정 요소"를 전수 조사하고 현재 세팅의 부족분·개선점을 점검 ② Claude Code 작업환경을 데스크톱 앱 중심에서 한 단계 발전 ③ 모바일에서 Claude Code를 조작·피드백받기(서버 PC 상시 가동 전제) ④ AI 영상제작·마케팅 등으로 기업을 운영·수익 창출하는 환경 조사.

도요 결정사항(2026-07-04):
- **우선순위**: 4개를 하나의 순차 로드맵으로 묶어서 진행
- **산출물**: 옵시디언 리포트로도 저장(권장) + 실행 계획
- **수익화 방향**: 영상 콘텐츠 자동화 + AI 마케팅 대행 + 아직 탐색 단계(넓게)

목표 결과물: (1) vault에 저장하는 종합 리포트 1개 (조사결과+갭진단+개선안+로드맵), (2) 트랙별 실제 실행(설정 변경·신규 에이전트·스킬·메모리 보강).

## Part A — 트랙 1: 모바일 + 작업환경 업그레이드 (즉시, 오늘 가능)

### A-1. Remote Control로 모바일 조작 (요청 핵심)

**결론: 별도 개발 불필요. Claude Code 정식 기능 `Remote Control`로 오늘 바로 됨.**

작동 원리: 서버 PC에서 세션 실행 → 폰/브라우저는 그 로컬 세션의 "창"일 뿐. **클라우드로 안 넘어가고 네 PC에서 그대로 실행**(파일·MCP·프로젝트 그대로). 인바운드 포트 안 열림, 전부 Anthropic API 경유 TLS.

전제 조건:
- 플랜: **Pro/Max/Team/Enterprise** (API 키 불가). 도요는 Claude 구독 사용 중 → 충족 예상
- 버전: **v2.1.51+** (`claude --version`으로 확인, 낮으면 업데이트)
- 인증: `claude` 실행 → `/login`으로 claude.ai 로그인 (`ANTHROPIC_API_KEY` 환경변수 있으면 해제)

설정 절차:
1. 서버 PC 프로젝트 폴더에서 `claude remote-control` 실행 (서버 모드) — 또는 세션 중 `/remote-control`(`/rc`)
2. 스페이스바로 QR 표시 → 폰 Claude 앱으로 스캔 (앱 없으면 세션 중 `/mobile`로 다운로드 QR)
3. 폰에서 앱 하단 **Code** 탭 → 세션 접속, 같은 대화·파일 조작
4. 모든 세션 자동 연결: `/config` → **Enable Remote Control for all sessions = true**
5. 푸시 알림(v2.1.110+): `/config` → **Push when Claude decides** / **Push when actions required** 켜기 → 긴 작업 끝나면 폰으로 알림

서버 상시가동 주의점:
- `claude` **프로세스가 살아있어야** 세션 유지 (터미널 닫으면 종료)
- 네트워크 10분 이상 끊기면 세션 타임아웃 → 재실행 필요
- 상시가동: 서버 PC에 터미널 하나를 `claude remote-control`로 계속 띄워둠 (절전 해제 설정 권장)

곁다리 옵션(참고, 지금 필수 아님):
- **Dispatch**: 폰 Claude 앱에서 작업을 메시지로 던지면 데스크톱 세션이 받아 처리
- **Claude Code on the web** (claude.ai/code): 클라우드에서 실행 — 로컬 세팅 없이 여러 작업 병렬. Remote Control과 반대(클라우드 실행)
- **Channels**: 텔레그램/디스코드/iMessage 연동 — 외부 이벤트에 반응

### A-2. 작업환경 업그레이드 (데스크톱 앱 → 하이브리드)

현재 병목: Claude 데스크톱 앱만 사용. 제안:
- **VS Code 확장 설치**: 코드/파일 편집 시 diff 시각화, `/rc`도 VS Code에서 지원 → 모바일 연결과 자연스럽게 이어짐
- **CLI 병행**: 서버 상시가동·스케줄·Remote Control 서버모드는 CLI가 본진
- 3층 구조 확립: **CLI(서버·자동화) + VS Code(편집) + 모바일(원격조작)** — 셋이 같은 로컬세션 공유

## Part C — 트랙 3: AI 수익화 환경 (영상 + 마케팅)

### C-1. 영상 콘텐츠 자동화 (관심 방향 ①)
이미 `콘텐츠-PPT` 파이프라인(줄이 대본 → 채린이 테마 → 판이 Marp PPT/PNG) 구축됨. 선우(Edge TTS)도 복직. **남은 건 PNG+음성 → MP4 합성 1단계.**

2026 영상 생성 도구 지형 (조사결과):
- **Google Veo 3.1** — 최고 품질, 프롬프트 정확도·네이티브 오디오·4K
- **Kling 3.0** — 가성비 최강, 멀티샷 스토리보드
- **Runway Gen-4/4.5** — 세밀한 제어(카메라·모션브러시·캐릭터 일관성)
- **HeyGen** — 아바타/토킹헤드, 175개국어 립싱크, $24/월 (마케팅·강의에 유리)
- ⚠️ **Sora 제외** — 2026-04-26 종료 예정

실행: 슬라이드형(현 파이프라인 확장, 무비용)을 먼저 완성 → 고품질 생성형(Veo/Kling)은 유료 검토. 신규 **미영이**(영상합성: PNG+TTS→ffmpeg→MP4+자막) 채용.

### C-2. AI 마케팅 대행 (관심 방향 ②)
2026 표준 모델(조사결과): 프로젝트비→**구독형/성과형(outcome-based)** 전환. 3분류: 태스크형 / 효율형 / **성장시스템형**(높은 리테이너·차별화). AI 콘텐츠 파이프라인은 인당 산출 60%↑, 솔로 창업에 유리(산출을 인건비에서 분리).

실행: 문이(블로그)+물색이(SEO)+캐순이(리서치)+손질이(검수) 팀이 이미 마케팅 콘텐츠 엔진. 여기에 멀티채널 리퍼포징(1글→블로그/링크드인/뉴스레터) 스킬을 얹으면 대행 서비스 최소단위 완성.

### C-3. 건설×AI 차별화 (넓은 탐색)
도요 본업(건축 PM)이 최대 차별점. 건설 AI 도구·논문·뉴스 자동수집 → vault 축적 스킬로 전문성 콘텐츠화. 장기 창업 자산.
