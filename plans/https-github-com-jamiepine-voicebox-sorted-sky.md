# 플랜: 유튜브/SNS 콘텐츠 자동화 파이프라인 (주제 → PPT → 영상)

## Context

도요님 요청: 주제나 블로그 글을 넣으면 AI가 자동으로 PPT(세련된 디자인)와 대본을 만들고,
최종적으로 영상(MP4)으로 출력하는 파이프라인 구축. 용도는 유튜브/SNS 콘텐츠.

현재 팀 공백:
- 대본 작성 에이전트 없음
- PPT 생성 에이전트 없음
- TTS 선우가 대기발령 중 → 대체 도구 검토 필요
- 슬라이드 + 음성 → MP4 합성 에이전트 없음

우선순위: PPT 먼저 → 이후 영상화.

---

## 파이프라인 전체 구조

```
[입력]
  주제 텍스트 or 기존 블로그 글
        │
        ▼
[1단계] 대본 작성 — 줄이 (신규)
  주제/글 → 유튜브/SNS용 나레이션 대본
  슬라이드 1장 = 대본 1단락 형식으로 분리
  산출물: 대본_YYMMDD_제목.md
        │
        ▼
[2단계] PPT 생성 — 판이 (신규)
  대본 섹션 → Marp 마크다운 → .pptx + 슬라이드 PNG
  채린이(디자인 시스템) 연계 → CSS 테마 적용
  산출물: .pptx / 슬라이드별 .png 이미지 시퀀스
        │
        ▼
[3단계] 영상화 — 추후 설계 (선우 대체 TTS 확정 후)
  TTS 음성 + 슬라이드 PNG → ffmpeg → MP4
```

## Phase 1: PPT 파이프라인 (즉시 구현)

### 신규 채용 — 줄이 (대본 작성 / 사원)

**이름 근거**: 줄(字·글줄) → 대본은 줄로 이루어짐 → 줄이

| 항목 | 내용 |
|------|------|
| 역할 | 주제/블로그 글 → 유튜브/SNS용 나레이션 대본 작성 |
| 입력 | 주제 텍스트 또는 기존 마크다운 파일 경로 |
| 출력 | `대본_YYMMDD_제목.md` — 섹션별 분리 |
| 저장 경로 | `{vault}\30_Areas\콘텐츠\대본\` |

**대본 출력 형식 (슬라이드 연동)**:
```
---슬라이드 1: 제목---
(나레이션 텍스트)

---슬라이드 2: 도입---
(나레이션 텍스트)
```

### 신규 채용 — 판이 (PPT 생성 / 사원)

**이름 근거**: 판(版·판형·레이아웃) → 슬라이드 판을 짜는 역할 → 판이

| 항목 | 내용 |
|------|------|
| 역할 | 대본 → Marp 마크다운 → .pptx + 슬라이드 PNG 출력 |
| 입력 | 줄이 대본.md + 채린이 CSS 테마 |
| 출력 | `.pptx`, `.pdf`, 슬라이드별 `.png` |
| 저장 경로 | `{vault}\30_Areas\콘텐츠\PPT\` |
| 도구 | Marp CLI (`npx @marp-team/marp-cli`) |

**디자인 전략**:
- 채린이가 `awesome-design-md`에서 DESIGN.md 수집 → Marp CSS 테마로 변환
- 기본 테마: `uncover` (심플·세련, Marp 내장)
- 커스텀 테마 파일 `marp-theme.css` 저장 후 재사용

**참고 영상**: [MARP 슬라이드 — AI로 3단계 레벨업 + 무료 템플릿 10개](https://www.youtube.com/watch?v=FHoDEQl6Pvg)
(채널: Daniel Zambrini | AI for Real Work)

**Marp 3단계 레벨 시스템 (영상 기반)**:
- **레벨 1**: AI에게 프롬프트로 Marp 마크다운 생성 요청 → 즉시 완성
- **레벨 2**: 데이터(표, 수치) 입력 + 실시간 빌드
- **레벨 3**: HTML 커스텀 + 자동화 시스템 (보고서 템플릿화)

→ 판이는 레벨 1~2 구현. 레벨 3(HTML 자동화)은 추후 확장.

**무료 템플릿**: https://github.com/Zambrini/MARP-Templates
→ 판이가 시작 시 이 레포에서 템플릿 다운로드해 활용.

**Marp CLI 실행 명령**:

# 플랜: 실시간 연동 운영 대시보드 (신규 — 스타듀밸리 폐기)

## Context

도요님 요청: Claude Code를 쓸 때 **지금 어떤 에이전트가 작업 중이고 어떤 에이전트가 대기 중인지,
내가 보유한 스킬이 무엇이고 무엇을 쓸 수 있는지** 한눈에 보는 실시간 연동 대시보드.
기존 스타듀밸리 대시보드는 폐기하고 처음부터 새로 만든다. 디자인도 깔끔하게.

기존의 한계: `agents.json`을 사람이 손으로 고쳐야 반영됨 → 가짜 실시간.
목표: 에이전트를 호출하는 순간 자동으로 상태가 바뀌는 진짜 실시간.

---

## 핵심: 실시간 연동 메커니즘 (Hook 기반)

Claude Code의 Hook이 에이전트 생명주기 이벤트를 잡아 상태 파일을 자동 갱신한다.

```
[Claude Code 세션]
  에이전트 spawn
      │
      ▼
  PreToolUse Hook (matcher: Task)  ──► state.json: 해당 에이전트 status="working"
      │
   (작업 수행)
      │
      ▼
  SubagentStop Hook  ──────────────► state.json: status="idle" + 로그 1줄 추가
      │
      ▼
[로컬 서버] state.json 변경 감지(watchdog)
      │
      ▼  SSE(Server-Sent Events) push
[브라우저 대시보드]  ──► 카드 색상/로그 즉시 갱신 (새로고침 불필요)
```

- Hook은 `.claude/settings.json`의 `hooks` 섹션에 등록 (현재 비어 있음)
- Hook 명령은 작은 Python 스크립트(`scripts/hook_update.py`) 호출 — stdin JSON에서
  tool_name·agent 정보 파싱 후 `state.json` 갱신
- SSE를 쓰는 이유: 상태는 서버→브라우저 단방향 push라 WebSocket보다 간단

## 화면 구성 — 탭(시트) 6개 (요구사항 6번)

좌측 사이드바 또는 상단 탭으로 전환. 다크 테마, 카드형, 플랫.

| 탭 | 내용 |
|----|------|
| 개요 | 요약 메트릭 + 직원 상태 그리드 + 한도 게이지 + 다음 예약 |
| 직원 | 12명 전체 카드, 클릭 시 상세 패널 (역할·스킬·처리건수·최근작업) — 요구4 |
| 스킬·명령 | 보유 스킬 목록 + **명령어 입력창** (요구2) |
| 일정 | 구글 캘린더 오늘/주간/월간/연간 뷰 (요구5) |
| 로그 | 전체 활동 로그 + 필터·검색 |
| 비용 | 토큰·비용 추적, 한도 현황 (요구1·7) |

### 요구사항별 반영

- **1. 한도 경고 실시간**: 상단 고정 게이지 바 — 주간 사용량 %, 리셋까지 남은 시간.
  임계 80% 넘으면 빨강. Hook이 매 에이전트 실행 시 사용 카운트 누적.
- **2. 대시보드에서 명령 입력**: 입력창 → 로컬 서버가 `claude --print -p "..."` 헤드리스 호출
  → 결과를 대시보드에 스트리밍 표시. **대시보드가 진짜 명령 센터가 됨.**
  (토큰 소모처는 여기뿐 — 아래 토큰 분석 참조)
- **3. 메트릭 카드 클릭 → 로그**: 작업중/대기/예약/오늘처리 카드 클릭 시 해당 항목만 필터된 로그.
- **4. 직원 클릭 → 상세**: 슬라이드 패널로 역할·담당 스킬·누적 처리건수·최근 작업 표시.
- **5. 구글 캘린더**: 캘린더 MCP 또는 Google Calendar API로 일정 추출.
  일/주/월/년 토글. 서버가 직접 API 호출 → 토큰 0.
- **6. 여러 시트**: 위 6개 탭.
- **7. 토큰 분석**: 아래 별도 섹션.

## 토큰 소모 분석 (요구사항 7)

| 기능 | 토큰 | 이유 |
|------|------|------|
| 정적 카탈로그 스캔 | **0** | 로컬 Python 파일 읽기 |
| Hook 상태 갱신 | **0** | Hook은 Python 실행, AI 호출 안 함 |
| SSE 실시간 push | **0** | 로컬 서버 |
| 캘린더 표시 | **0** | Google API 직접 호출 |
| 한도·비용 표시 | **0** | 로컬 카운터 누적 |
| **대시보드 명령 실행** | **소모** | `claude --print` 호출 — 명령당 1세션 |

**결론**: 모니터링·표시 기능은 전부 토큰 0 → 맘껏 추가.
토큰은 "대시보드에서 AI에게 일 시킬 때"만 발생 → 실행 기능만 신중히 설계.
**추가 권장**: 표시계 전부. **삭제 권장**: 없음 (실행 기능은 사용량에 비례하므로 유지하되 확인 다이얼로그).

---

## 1. 네이밍 기준 (Name Convention)

### 원칙
1. **실제 한국 이름이어야 한다** — 사람 이름처럼 들릴 것. 외래어 음차 금지.
2. **역할을 한자·순우리말로 연상 후 비틀기** — 역할을 한자어나 순우리말로 바꾸고, 거기서 한 글자·음소를 이름으로 가져온다.
3. **직관적 연결 금지** — 이름만 보고 바로 역할을 알면 안 됨. 알고 나면 "아, 그렇구나" 느낌이 나야 함.
4. **2~3음절 + 이(이름 어미)** 형태 권장 — 강이, 문이, 재이, 옥순이, 새벽이 등.

### 기존 직원 네이밍 검토
| 이름 | 역할 연상 | 판정 |
|------|-----------|------|
| 말숙이 | 역할과 무관, 총괄 비서 느낌 | ✅ 의도적 촌스러움 |
| 옥순이 | 옥(玉·정리), 순(順·규칙) → vault 정리 | ✅ |
| 새벽이 | 새벽에 일함 → 너무 직관적 | △ 허용 |
| 강이 | 강의(講義) → 유튜브 강의 | ✅ |
| 문이 | 문(文) → 글쓰기 | ✅ |
| 캐순이 | 캐다·캐묻다 → 리서치 | ✅ |
| 물색이 | 物色 → 키워드 탐색 | ✅ |
| 손질이 | 손질하다 → 검수·다듬기 | ✅ |
| 선우 | 성우(聲優) 비틀기 | ✅ |
| ~~파이이~~ | Fire→파이어 음차 | ❌ 이름 아님 |
| ~~인세이~~ | Insane→인세인 음차 | ❌ 이름 아님 |

### 향후 이름 짓기 프로세스
1. 역할의 핵심 동작을 한자어로 찾는다 (채용=採用, 크롤=탐색探索, 분석=析)
2. 그 한자에서 음을 가져와 실제 한국 이름으로 조합한다
3. 이름 자체가 자연스러운지 확인 (실제 사람 이름처럼 들리는가)

## 각 도구 특성 (학습노트 기반)

### Firecrawl
- **방식**: 유료 API 서비스. API Key 필요. MCP 공식 지원.
- **강점**: JS 렌더링, 봇 탐지 우회, 마크다운 변환 자동화. 깔끔한 출력.
- **약점**: 유료 (크레딧 소모), 외부 API 의존.
- **사용법**: `firecrawl_scrape(url)` → 마크다운 반환

### Insane-search
- **방식**: 로컬 실행 Python 라이브러리. 무료.
- **강점**: 비용 없음, 공개 페이지 크롤링에 특화.
- **약점**: JS 렌더링 제한, 봇 탐지에 취약할 수 있음.
- **사용법**: Python 스크립트로 실행

# (이전 플랜) 문이 확장 — 사진 없는 주제 기반 자동 블로그 + 서브에이전트 팀

## Context

도요 요청: 문이가 "내가 준 사진"에만 의존하지 말고, **사진 없이 주제만으로 AI가 자동으로 글을 쓰는 모드**를 갖췄으면 함. 리서치는 자동 웹검색.

조사 결론 (2026-06):
- **독자는 사진 너머 정보를 원함**: 위치·영업시간·주차·가격·웨이팅 등 실용정보가 체류시간·검색유입을 좌우. → 사진만으로는 한계.
- **멀티에이전트 글쓰기 표준 = 리서처 → 키워드/전략 → 작가 → 에디터(사실검증)**. 분업이 환각을 줄이고 품질을 올림. 핵심: "사실 검증을 글쓰기 전에 끝낸다".

### 중요한 전제 (저품질·신뢰 회피)
- 사진 없는 자동 글은 **체험기(다녀온 후기) 금지** — 안 해본 경험을 지어내면 네이버 저품질 분류 + 신뢰 붕괴.
- 대신 **정보·정리형 글**로 한정하면 완전 자동화 가능하고 진정성도 유지.
- 글 성격을 정직하게: "정보 정리" 글임을 톤으로 드러내고, 가능하면 도요 관점(건축PM·AI 전환)을 얹어 차별화.

### 자동화 가능한 글 유형 (사진 불필요)
1. **정보 큐레이션** — 예: "2026 육아휴직·아빠 출산휴가 제도 총정리"
2. **비교·랭킹** — 예: "신생아 분유 5종 비교", "용인 가볼만한 곳 정리"
3. **하우투·가이드** — 예: "건설 현장에서 엑셀 자동화 시작하는 법"
4. **트렌드·뉴스 해설** — 시사·기술 이슈 쉽게 풀기
5. **도요 전문 칼럼** — 건축 PM × AI 전환 (개인 브랜딩, 가장 차별화)

   - 산출물: 구조화된 "자료팩"(사실 + 출처 URL) → 문이에게 전달

2. **물색이** (物色) — 키워드/SEO 에이전트
   - 네이버 연관검색어·인기 키워드·상위 노출 글 구조 조사
   - 제목(질문형 옵션)·소제목·태그 후보 제안 (C-Rank/DIA)
   - 산출물: 키워드 세트 + 추천 제목/구조

3. **손질이** (손질하다) — 에디터·검수 에이전트
   - **발행 전 게이트**: 문이 초안을 자료팩과 대조해 사실 검증 (미출처 주장 플래그)
   - 맞춤법·가독성·저품질 4종 검열(상업성 키워드·중복표현·중복이미지·과장)
   - 통과해야 저장 진행 ("검증 green 전엔 미완료")
