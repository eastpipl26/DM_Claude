# 워치리스트 대시보드 — 로컬 HTML, 필터링 가능

## Context

지금까지는 매주 md 리포트를 텍스트로만 봤는데, 단지·타입·기간별로 잘라서 보려면 매번 CSV를 다시 조회해야 했다. 로컬 HTML 파일 하나로 워치리스트 데이터를 시각화하고, 브라우저에서 바로 필터링(단지·평형·기간·공구)할 수 있게 만든다.

인터넷 접속 없이도 열리도록(파일 더블클릭 → file:// 로 바로 열림) 외부 CDN 없이 순수 HTML/CSS/JS로 만들고, 데이터는 빌드 시점에 JSON으로 통째로 파일 안에 심는다(서버 불필요).

## 접근 방식

**신규 스크립트 `{re_scripts}\build_dashboard.py`**

1. `track_watchlist.py`의 `WATCHLIST`, `prev_week_rows`, `compare_to_prev`를 그대로 import해서 재사용 — 이상징후 판정 로직 중복 작성 안 함
2. 데이터 소스:
   - `실거래_매매.csv` → 워치리스트 8개 단지만 필터, 전용면적을 반올림해 `타입`(59/64/74/84㎡ 등) 필드 생성
   - `워치리스트_주간스냅샷.csv` → 매물비율·증감률 주차별 이력 (이미 누적 중)
3. 위 데이터를 JSON으로 직렬화해 HTML `<script>` 태그 안에 인라인으로 심음
4. 클라이언트 사이드(순수 JS)로:
   - **필터**: 단지명(다중선택 체크박스), 타입, 기간(최근1년/3년/5년/전체 토글), 공구(심층/대장)
   - **실거래가 추이 차트**: 선택된 단지·타입의 거래금액을 시간순 라인차트(SVG 직접 렌더링, 라이브러리 없음)
   - **매물비율·증감률 추이**: 워치리스트 스냅샷 주차별 라인차트
   - **단지별 비교표**: 현재 매물비율·증감률·최근거래가·고점대비%(있으면) 컬럼, 헤더 클릭으로 정렬
   - **이상징후 배너**: 최신 주차 기준 이상징후 있으면 상단에 빨간 배너로 강조
5. 출력: `{real_estate}\대시보드.html` (트래커 하위 폴더가 아니라 부동산 폴더 바로 아래 — 매주 새로 생기는 리포트와 달리 이건 계속 갱신되는 단일 파일)

**`run_weekly.py`에 5번째 단계로 추가**: track_watchlist.py 다음에 build_dashboard.py 실행 → 매주 자동 갱신

## 신규/수정 파일

| 파일 | 내용 |
|---|---|
| `{re_scripts}\build_dashboard.py` | 대시보드 HTML 생성 (신규) |
| `{re_scripts}\run_weekly.py` | STEPS에 build_dashboard.py 추가 (기존 파일 수정) |
| `{real_estate}\대시보드.html` | 산출물 (신규, 매주 덮어쓰기) |
