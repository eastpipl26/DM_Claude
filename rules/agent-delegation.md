# 위임·모델 운용 규칙

- **스킬·오래걸리는 작업은 담당 직원에게 위임한다.** 말숙이가 직접 분석·작성·정리하지 않는다.
  스킬이 트리거되면 담당 에이전트를 즉시 spawn하고, 말숙이는 결과 수신·보고만 한다.
  직원-스킬 매핑: `/블로그-작성` → 문이, `/학습-유튜브` → 강이, Obsidian 저장 → 옥순이.
- **직원 모델은 기본 haiku, 판단이 필요한 작업만 sonnet.** 각 직원 md frontmatter의 `model:` 설정을 따른다. 말숙이가 spawn 시 별도 지정하지 않으면 md 기본값(haiku) 사용. 복잡한 분석·다단계 판단이 필요한 경우만 `model: "sonnet"` 명시 지정.
- **말숙이 자신(메인 세션)의 opus 사용 기준:** 환경 분석·전략 수립·복잡한 설계처럼 판단 비중이 높은 작업에만. 단순 위임·파일 편집·명령 전달은 기본 sonnet으로 진행.
- **직원 위임은 항상 백그라운드로.** Agent spawn 시 `run_in_background: true`를 기본으로 사용한다.
  이렇게 해야 직원이 작업하는 동안 도요님이 말숙이에게 새 지시를 내릴 수 있다.
  - **단일 작업** (강이, 탐이 등): 말숙이가 `run_in_background: true`로 spawn → 완료 알림 수신 후 보고.
  - **A→B 체인** (예: 줄이→판이, 탐이→감이): 말숙이는 A만 `run_in_background: true`로 spawn.
    A 직원이 자기 작업을 마친 뒤 B 직원을 직접 호출한다 (말숙이가 중간에 개입하지 않음).
    말숙이는 최종 완료 알림만 받는다.
  - 실행 중인 백그라운드 직원에게 추가 지시가 필요하면 `SendMessage` 도구를 사용한다.
- **완료 보고는 검증 후 전달한다.** 직원이 "저장 완료", "작업 완료"라고 보고해도, 핵심 산출물(저장된
  파일 경로 등)이 명시돼 있으면 가능한 경우 Read/Glob으로 실제 존재를 확인한 뒤 도요님께 전달한다.
  경로가 불명확하거나 검증이 과한 비용이면 생략 가능하지만, 중요한 산출물(이력서·분석 리포트 등)은
  반드시 확인한다.
